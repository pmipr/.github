# 🚀 PMI Paraná Chapter - Diretoria de TI (DTI)

Bem-vindo(a) ao ecossistema de desenvolvimento e gestão de projetos do **PMI Paraná**. Este espaço é a nossa central de colaboração técnica, focado em transformar a nossa DTI numa área de **Arquitetura de Soluções**.

---

## 🎯 O Nosso Propósito
O GitHub do PMI PR não é apenas um repositório de código. É a plataforma onde materializamos o valor estratégico para o Chapter através de três pilares:

1.  **Data Products:** Construção da nossa *Single Source of Truth* (API Global + Database).
2.  **Eficiência de Processos:** Mapeamento e automação de fluxos (Market Scan -> UAT).
3.  **Segurança e Governança:** Implementação da nossa PSI (Fortaleza Workspace).

---

## 🛠️ Como Trabalhamos (Agile & Open Source Interno)

Adotamos metodologias ágeis para garantir entregas incrementais e de alto valor.
* **Gestão:** Utilizamos o GitHub Projects e o Monday.com em sincronia.
* **Documentação:** "Code is documentation". Todo projeto deve ter o seu próprio `README.md` detalhado.
* **Transparência:** Todos os voluntários têm visibilidade dos projetos ativos para incentivar a colaboração entre áreas.

---

## 🛡️ Regras de Ouro (Segurança & PSI)

De acordo com a nossa **Política de Segurança da Informação (PSI)**, todos os membros devem seguir estas diretrizes:

* **Zero Secrets:** Nunca faça commit de chaves de API, senhas ou tokens. Utiliza sempre *GitHub Secrets* ou variáveis de ambiente.
* **RBAC:** O acesso é baseado na função. Se precisares de permissões adicionais, solicita via chamado.
* **Branching:** Não faça push direto na `main`. Utiliza *Pull Requests* (PRs) para que outros membros possam rever o teu trabalho.
* **Dados Sensíveis:** Não armazene dados pessoais de filiados (LGPD) em repositórios públicos ou ficheiros de texto simples.
