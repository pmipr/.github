# 🚀 Bem-vindo ao GitHub do PMI Paraná Chapter!

Bem-vindo(a) à central de gerenciamento de projetos e colaboração de **todas as diretorias** do PMI PR. 

Esqueça a ideia de que o GitHub é apenas para escrever código! Nós configuramos este espaço para ser o nosso escritório virtual. Aqui, gerenciamos nossas iniciativas, acompanhamos tarefas em quadros Kanban e centralizamos a documentação do Chapter, garantindo transparência e agilidade para todos os voluntários.

---

## 🎯 Por que estamos usando o GitHub?

* **Visibilidade 360º:** Todos sabem o que está acontecendo no Chapter.
* **Gestão Ágil:** Quadros visuais (Kanban) para acompanhar o progresso das entregas.
* **Histórico e Memória:** Fim das informações perdidas em grupos de WhatsApp. Tudo fica documentado e rastreável.
* **Segurança:** Acessos controlados e alinhados com a nossa Política de Segurança da Informação (PSI).

## 📖 Dicionário Básico (Para não-técnicos)

Se você nunca usou o GitHub, não se preocupe! Aqui estão os 3 termos que você precisa conhecer:
1. **Repositório (Repo):** É a "pasta" do seu projeto. Onde ficam os arquivos, documentos e o histórico de conversas daquela iniciativa.
2. **Issue:** É uma "Tarefa". Se você precisa que alguém faça algo, ou quer registrar uma ideia, você abre uma Issue.
3. **Project:** É o seu "Quadro Kanban" (estilo Trello ou Planner), onde você organiza as Issues em colunas (A Fazer, Em Andamento, Concluído).

## 🛠️ Tutorial: Como gerenciar seu projeto aqui

### Passo 1: Criando o Repositório do seu Projeto
*Apenas diretores e líderes de projeto têm permissão para criar repositórios.*
1. Na página inicial do PMI PR, clique no botão verde **"New"** (Novo).
2. Dê um nome claro (ex: `MKT-Evento-Anual-2026`).
3. Mantenha a visibilidade como **Private** ou **Internal** (nunca Public, para proteger nossos dados).
4. Marque a caixa **"Add a README file"** e clique em **Create repository**.

### Passo 2: Criando Tarefas (Issues)
1. Dentro do seu novo repositório, clique na aba **"Issues"** e depois no botão verde **"New Issue"**.
2. Coloque um título claro e descreva o que precisa ser feito.
3. No menu lateral direito:
   * **Assignees:** Marque o voluntário responsável por executar a tarefa.
   * **Labels:** Adicione etiquetas (ex: `urgente`, `financeiro`, `marketing`).

### Passo 3: Criando seu Quadro Kanban (Projects)
1. Vá até a aba **"Projects"** na barra superior da organização.
2. Clique em **"New project"** e escolha o modelo **"Board"** (Quadro).
3. Dê um nome ao seu quadro. 
4. Agora, basta clicar em "+ Add item" nas colunas para puxar as *Issues* que você criou no Passo 2!

## 🛡️ Regras de Ouro e Compliance (PSI LGPD)

Como lidamos com informações do Chapter, todos os voluntários devem seguir estas diretrizes básicas de segurança:

* **🚫 Zero Dados Pessoais:** Não suba planilhas com CPFs, telefones ou dados sensíveis de filiados aqui. O GitHub é para gerenciar *processos*, não para armazenar banco de dados de pessoas.
* **🚫 Zero Senhas:** Nunca escreva senhas de ferramentas do Chapter em Issues ou documentos de texto.
* **✅ Clareza e Respeito:** A comunicação nas Issues deve ser sempre profissional, clara e colaborativa.
